# Чек-лист перед кодированием

## Обязательно проверить:
- [ ] Есть прямое "Делай" / "Вносим" / "Реализуй" от пользователя?
- [ ] Ясны входные данные (что приходит)?
- [ ] Ясны выходные данные (что должно получиться)?
- [ ] Есть fallback / план Б если не сработает?
- [ ] Это не нарушает паттерны из SOUL.md?

## Если НЕТ хотя бы одного пункта:
→ НЕ открывать редактор
→ Запросить согласование через формат:
    ПРОБЛЕМА:
    ВАРИАНТЫ:
    РЕКОМЕНДАЦИЯ:
    ОЖИДАЮ:

## Триггеры (когда можно кодить):
✅ "Давай вносить изменения"
✅ "Делай"
✅ "Начинай"
✅ "Реализуй"

## Триггеры (когда НЕЛЬЗА кодить):
❌ "Давай подумаем"
❌ "Обсудим"
❌ "Как поступим?"
❌ "Что думаешь?"
❌ "Может..."

## Примеры "мелочей" (можно без обсуждения):
- Добавление logger.info для отладки
- Исправление опечаток
- Добавление комментариев
- Переименование переменных для ясности

## НЕ "мелочи" (только после согласования):
- Новые файлы
- Изменение логики
- Новые API endpoints
- Удаление кода
- Изменение базы данных
